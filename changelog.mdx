## Recent Updates

Stay informed about all changes to Angga. You can subscribe to release notifications via the dashboard at `https://dashboard.example.com/notifications`.

<Update label="2024-10-15" description="v1.2.0" tags={["feature", "improvement"]}>

## New Features

- Added real-time collaboration for documentation editing, allowing multiple users to work simultaneously.
- Introduced version history with diff views to compare changes easily.

## Improvements

- Enhanced search functionality with fuzzy matching and syntax highlighting.
- Optimized performance for large documentation sites, reducing load times by `>30%`.

## Bug Fixes

- Fixed rendering issues with nested lists in MDX components.
- Resolved authentication token refresh problems during long sessions.

</Update>

<Update label="2024-09-20" description="v1.1.0" tags={["feature", "bugfix"]}>

## New Features

- Integrated custom themes with support for brand colors like `#3B82F6`.
- Added export options for PDF and HTML from your documentation spaces.

## Bug Fixes

- Corrected table alignment problems in responsive views.
- Patched security vulnerability in webhook handling (CVE-2024-XXXX).

## Backward Compatibility

All API endpoints remain unchanged. Update your integrations without modifications.

</Update>

<Update label="2024-08-10" description="v1.0.0" tags={["breaking", "feature"]}>

## Initial Release

- Core documentation management with MDX support.
- User authentication via API keys and OAuth.
- Sidebar navigation and search across pages.

## Breaking Changes

- Removed legacy YAML frontmatter parser; migrate to standard format.

To upgrade, run:

````bash
npm update angga@latest
````

</Update>

## Upgrade Guide

Follow these steps to upgrade Angga safely.

<Steps>
  <Step title="Backup" icon="database">

  Export your documentation:

  ```bash
  angga export --space my-space --format json
  ```

  </Step>
  <Step title="Update Package" icon="download">

  ```bash
  npm install angga@latest
  ```

  Review [release notes](https://dashboard.example.com/releases) for each version.

  </Step>
  <Step title="Test Locally" icon="play">

  Start dev server:

  ```bash
  angga dev
  ```

  Verify all pages render correctly.

  </Step>
  <Step title="Deploy" icon="rocket">

  Push to production:

  ```bash
  angga deploy
  ```

  </Step>
</Steps>

<Callout kind="alert">
  Always test in staging before production. Breaking changes are noted in each release.
</Callout>
